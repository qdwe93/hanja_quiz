# 한자 데이터 명세

## 1. 범위와 결론

이 데이터셋은 전국한자능력검정시험 배정한자 중 **7급·준6급·6급에서 새로 추가되는 200자**만 담는다. 누적 범위의 8급 50자와 준7급(공식 표기 7급II) 50자는 포함하지 않는다.

2026-07-11에 원문과 공식 급수표를 확인한 결과, 실제 신습 수량은 다음과 같다.

| 앱 등급 | 공식 등급 표기 | 누적 읽기 수 | 직전 급수 대비 신습 수 | 데이터 수 |
|---|---:|---:|---:|---:|
| 7급 | 7급 | 150 | 150 - 100 = 50 | 50 |
| 준6급 | 6급II | 225 | 225 - 150 = 75 | 75 |
| 6급 | 6급 | 300 | 300 - 225 = 75 | 75 |
| **합계** |  |  |  | **200** |

프로젝트 프롬프트의 예상값인 `7급 50 + 준6급 50 + 6급 100`은 원문의 등급 경계와 일치하지 않는다. 수량을 임의로 맞추면 25자의 등급이 바뀌므로, 원문과 한국어문회 공식 누적 수치가 함께 지지하는 **50 + 75 + 75**를 채택했다.

## 2. 출처와 수집 방법

### 주 출처

- [전국한자능력검정시험/배정한자](https://namu.wiki/w/%EC%A0%84%EA%B5%AD%ED%95%9C%EC%9E%90%EB%8A%A5%EB%A0%A5%EA%B2%80%EC%A0%95%EC%8B%9C%ED%97%98/%EB%B0%B0%EC%A0%95%ED%95%9C%EC%9E%90)
- 접근 확인에 사용한 [동일 문서 미러](https://namu.moe/w/%EC%A0%84%EA%B5%AD%ED%95%9C%EC%9E%90%EB%8A%A5%EB%A0%A5%EA%B2%80%EC%A0%95%EC%8B%9C%ED%97%98/%EB%B0%B0%EC%A0%95%ED%95%9C%EC%9E%90) — 확인 당시 표시된 최근 수정 시각: 2026-03-28 22:07:12

주 출처는 각 급수에서 새로 추가되는 글자만 열거하며, 한자와 한국어문회 대표훈음을 함께 제공한다. `data/hanja.json`의 배열 순서는 주 출처의 급수별 열거 순서를 따른다.

### 교차 확인

- [사단법인 한국어문회 급수배정 안내](https://www.hanja.re.kr/kccpt/exam/levelConfirm.do)
- 같은 페이지가 제공하는 `배정한자 전체 대표훈음 부수 음표 명기본(xls).xls`

공식 급수표의 누적 읽기 수 `7급II 100 → 7급 150 → 6급II 225 → 6급 300`으로 신습 수량 50·75·75를 확인했다. 공식 전체 배정표에서 대표훈음 표기와 자형도 표본 대조했다. 원문 각주 번호는 훈음 자체가 아니므로 `sourceLabel`에서 제외했다.

## 3. JSON 스키마

`data/hanja.json`은 객체 200개로 이루어진 JSON 배열이다.

| 필드 | 타입 | 규칙 |
|---|---|---|
| `id` | `string` | 안정적인 고유 ID. `g7-001`, `g6p-001`, `g6-001` 형식 |
| `hanja` | `string` | 주 출처 자형을 보존한 한자 한 글자 |
| `eum` | `string[]` | 독음 목록. `hun`과 같은 인덱스가 한 훈음 쌍 |
| `hun` | `string[]` | 훈 목록. 슬래시는 같은 독음의 병렬 대표훈을 보존 |
| `eumhun` | `string` | 화면과 기본 정답에 쓰는 첫 번째 대표훈음. 장단음 기호만 제거 |
| `grade` | `"7급" \| "준6급" \| "6급"` | 해당 글자가 처음 추가되는 급수 |
| `source` | `string` | 항목을 추적할 수 있는 주 출처 URL |
| `sourceLabel` | `string` | 장단음과 복수 훈음을 보존한 원문 대표훈음. 각주 번호는 제외 |

예시:

```json
{
  "id": "g7-005",
  "hanja": "洞",
  "eum": ["동", "통"],
  "hun": ["골", "밝을"],
  "eumhun": "골 동",
  "grade": "7급",
  "source": "https://namu.wiki/w/...",
  "sourceLabel": "골 동ː | 밝을 통ː"
}
```

## 4. 정규화 규칙

1. **자형:** 주 출처의 BMP 한자 한 글자를 그대로 저장한다. 예를 들어 `強`, `樂`, `畫`를 임의로 `强`, `樂`, `畵`로 바꾸지 않는다. 글꼴에 따른 자형 차이는 데이터 변경 사유가 아니다.
2. **원문 보존:** `sourceLabel`에는 `ː`, `(ː)`, 슬래시, 복수 훈음 구분자 ` | `를 보존한다. 원문 설명을 가리키는 각주 번호만 뺀다.
3. **복수 독음:** ` | `로 구분된 훈음을 `eum`과 `hun`의 평행 배열로 저장한다. 한 독음 안에서 슬래시로 병기한 대표훈은 한 문자열로 유지한다.
4. **화면 표기:** `eumhun`은 첫 번째 훈음에서 장음 `ː` 또는 허용 장음 `(ː)`만 제거한 `훈 음` 형식이다. 초등 학습 화면에서는 장단음 차이를 정답 구별 기준으로 사용하지 않는다.
5. **등급:** 누적 응시 범위가 아니라 그 글자가 처음 추가되는 등급을 저장한다.
6. **순서와 ID:** 원문의 등급별 순서를 보존하고 1부터 3자리 일련번호를 부여한다. 데이터 순서가 바뀌더라도 이미 배포한 ID는 재사용하지 않는다.

## 5. 복수 훈음과 예외

복수 독음을 가진 글자는 8자다.

| 한자 | 원문 대표훈음 | 앱 기본 표기 |
|---|---|---|
| 洞 | 골 동ː \| 밝을 통ː | 골 동 |
| 便 | 편할 편(ː) \| 똥오줌 변 | 편할 편 |
| 讀 | 읽을 독 \| 구절 두 | 읽을 독 |
| 樂 | 즐길 락 \| 노래 악 \| 좋아할 요 | 즐길 락 |
| 省 | 살필 성 \| 덜 생 | 살필 성 |
| 度 | 법도 도(ː) \| 헤아릴 탁 | 법도 도 |
| 行 | 다닐 행(ː) \| 항렬 항 | 다닐 행 |
| 畫 | 그림 화ː \| 그을 획 | 그림 화 |

`主`, `反`, `區`, `李`, `別`, `使`처럼 슬래시가 있지만 독음이 하나인 항목은 대표훈의 병기이므로 배열 원소도 하나다.

`者`의 `놈 자`, `利`의 `이할 리`처럼 현대 학습자에게 낯설 수 있는 표현도 원문 대표훈을 바꾸지 않았다. 쉬운 풀이를 추가하려면 원본 필드를 덮어쓰지 말고 별도 설명 필드를 도입해야 한다.

## 6. 퀴즈 보기 생성 규칙

- 기본 정답 문자열은 `eumhun`이다.
- 오답 후보는 선택 범위 안에서 뽑되, 정답 항목 자신과 같은 `eumhun`을 가진 항목을 제외한다.
- 더 안전하게 하려면 각 항목의 모든 `hun[i] + " " + eum[i]` 조합을 비교하여, 정답 글자의 어느 독음과도 같은 후보를 제외한다.
- 한 문제의 보기 문자열은 `Set` 기준으로 정확히 네 개여야 하며, 섞은 뒤에도 정답 문자열은 정확히 한 번만 있어야 한다.
- 현재 200자에는 기본 `eumhun` 중복도, 모든 훈음 쌍 전체에서의 동일 문자열 중복도 없다. 이 사실에 의존하지 말고 생성 단계에서도 위 방어 규칙을 유지한다.

## 7. 검증 결과

수집 직후와 JSON 저장 후 다음 조건을 확인한다.

| 검사항목 | 결과 |
|---|---:|
| JSON 파싱 | 통과 |
| 전체 항목 수 | 200 |
| 7급 수 | 50 |
| 준6급 수 | 75 |
| 6급 수 | 75 |
| 고유 `id` 수 | 200 |
| 고유 `hanja` 수 | 200 |
| 빈 필수 필드 | 0 |
| 허용되지 않은 등급 | 0 |

| U+FFFD 또는 대표적인 UTF-8 깨짐 패턴 | 0 |

재현용 최소 검증 예시는 다음과 같다.

```powershell
node -e "const fs=require('fs');const d=JSON.parse(fs.readFileSync('data/hanja.json','utf8'));const c=Object.groupBy?Object.groupBy(d,x=>x.grade):d.reduce((a,x)=>((a[x.grade]??=[]).push(x),a),{});console.log(d.length,Object.fromEntries(Object.entries(c).map(([k,v])=>[k,v.length])),new Set(d.map(x=>x.id)).size,new Set(d.map(x=>x.hanja)).size)"
```

프로젝트의 자동 테스트에서는 위 수량 검사에 더해 필드 타입, 허용 등급, 깨진 문자, 평행 배열, 보기 유일성까지 검사해야 한다.

## 8. 알려진 차이와 유지보수

- 프롬프트의 등급별 예상 수량과 원문 수량이 다르다. 이 문서와 데이터는 원문 기준 50·75·75를 따른다.
- 한국어문회 공식 페이지는 준급을 `II`로 표기하지만 앱에서는 사용자 요구에 맞춰 `준6급`으로 표시한다.
- 주 출처나 한국어문회 배정표가 바뀌면 수집 날짜, 원문 수정 시각, 수량 차이를 먼저 기록하고 검증을 다시 실행한다. 기존 ID를 조용히 다른 글자에 재할당하지 않는다.
