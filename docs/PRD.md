# 한자랑 제품 요구사항 정의서(PRD)

> 문서 상태: 25자 세트 학습 개선 반영
> 작성일: 2026-07-11
> 대상 릴리스: MVP 1.0

## 1. 제품 개요

한자랑은 전국한자능력검정시험의 7급·준6급·6급 배정한자 200자를 두 가지 짧은 게임으로 익히는 한국어 웹앱이다. 사용자는 한자 카드와 음훈 카드를 연결하는 **짝맞추기**와, 한자를 보고 네 개의 음훈 중 정답을 고르는 **4지선다 퀴즈**를 이용한다.

원문의 신습자 수는 **7급 50자, 준6급 75자, 6급 75자**로 확인되며 합계는 200자다. 초기 프롬프트의 `50/50/100`은 예상치였으므로 제품과 검증 기준에는 원문 수량을 적용한다. 자세한 근거와 정규화 규칙은 [DATA_SPEC.md](./DATA_SPEC.md)를 따른다.

## 2. 해결하려는 문제

초등학생이 한자의 모양과 한국어 음훈을 함께 외우려면 반복 노출뿐 아니라 직접 구별하고 연결하는 활동이 필요하다. 기존의 한자 단독 플래시카드는 이미 대체 수단이 있고, 정답 피드백이 늦거나 화면에 한 번에 너무 많은 정보를 보여 주면 학습 집중도가 떨어진다.

이 제품은 다음 문제를 해결한다.

- 한자와 음훈을 능동적으로 연결할 간단한 연습 수단이 부족하다.
- 짧은 시간에도 시작하고 끝낼 수 있는 명확한 학습 단위가 필요하다.
- 학습 범위와 최근 결과를 기억하면서도 회원가입이나 개인정보 입력은 피해야 한다.
- 모바일, 키보드, 보조기술 사용자도 같은 핵심 흐름을 완료할 수 있어야 한다.

## 3. 대상 사용자와 사용 맥락

### 주 사용자

- 초등학교 6학년 학생
- 7급부터 6급까지 한자를 학습하거나 복습하는 사용자
- 스마트폰, 태블릿 또는 가정용 PC에서 5~10분 단위로 연습하는 사용자

### 보조 사용자

- 학생에게 학습 범위를 지정하거나 결과를 함께 확인하는 보호자·교사

### 대표 사용자 요구

1. 설명서를 읽지 않고도 등급과 게임을 고를 수 있다.
2. 한 문제 또는 한 라운드의 진행 정도를 즉시 알 수 있다.
3. 왜 맞거나 틀렸는지 음훈과 함께 확인할 수 있다.
4. 실수로 빠르게 누르더라도 게임 상태가 망가지지 않는다.
5. 새로고침 후에도 최근 선택과 결과 요약을 확인할 수 있다.

## 4. 제품 목표

- 검증된 200자를 25자 단위의 여덟 개 학습 세트로 정확히 제공한다.
- 첫 방문 사용자가 30초 안에 학습 범위와 게임 모드를 선택해 첫 행동을 할 수 있게 한다.
- 짝맞추기와 4지선다 모두 한 세트를 끝까지 오류 없이 완료할 수 있게 한다.
- 정답·오답·진행 상태를 색상에 의존하지 않고 명확히 전달한다.
- 별도 계정과 서버 저장 없이 기기 안에서 최근 학습 맥락을 복원한다.
- 320px 너비의 모바일부터 데스크톱까지 핵심 콘텐츠가 잘리거나 가로 스크롤되지 않게 한다.

## 5. 비목표

MVP에는 다음을 포함하지 않는다.

- 한자만 앞뒤로 넘기는 일반 플래시카드
- 회원가입, 로그인, 사용자 간 순위, 교사 관리 화면
- 서버 데이터베이스, 기기 간 동기화, 클라우드 백업
- 결제, 광고, 상품 추천, 소셜 공유
- 필기 인식, 발음 음원, 자동 문장 생성
- 전체 학습 이력 분석이나 개인정보 기반 추천
- 데이터 원문을 앱에서 직접 편집하는 관리자 기능

| 소리 | 제공하지 않음 | 핵심 학습 가치와 접근성에 집중 |

## 7. 정보 구조와 사용자 흐름

### 7.1 첫 학습

1. 사용자가 시작 화면에서 앱 목적과 두 학습 모드를 확인한다.
2. `7급-1`부터 `6급-3`까지 여덟 개 세트 중 하나를 선택한다.
3. 선택 세트의 25자 수가 즉시 갱신된다.
4. `짝맞추기 시작` 또는 `4지선다 시작`을 선택한다.
5. 게임을 진행하고 완료 결과를 확인한다.
6. 같은 범위의 새 세트, 다시 하기, 다른 게임, 시작 화면 중 하나를 선택한다.

### 7.2 재방문

1. 앱이 손상 여부를 확인한 로컬 기록을 읽는다.
2. 마지막 학습 세트와 간단한 누적·최근 결과를 보여 준다.
3. 사용자는 그대로 시작하거나 범위를 바꾼다.

### 7.3 오류 복구

1. 저장 데이터가 없거나 잘못되면 안전한 초기값으로 시작한다.
2. 선택 범위의 유효 데이터가 부족하면 게임 진입을 막고 한국어 오류 안내와 홈 이동 동작을 제공한다.
3. 예기치 않은 화면 오류가 발생해도 다시 시작할 수 있는 경로를 제공한다.

| ID | 요구사항 |

| MATCH-03 | 일반 구간은 맞는 4쌍과 짝이 없는 4장을 유지하며, 두 장 선택으로 비교 중에는 추가 선택을 잠근다. |

| MATCH-07 | 대기 풀이 4장 이하이면 한자·음훈을 한 장씩 임의 공급하며, 전체 남은 카드가 12장이 되면 추가 공급을 멈춘다. |
| MATCH-08 | 맞춘 쌍/25쌍과 시도 횟수를 표시하고, 전체 오답 시도 수와 중복 없는 오답 한자·음훈 목록을 결과에 표시한다. |
| MATCH-09 | 25쌍을 모두 맞추면 완료 상태로 전환하고 완료 횟수를 안전하게 저장한다. |
| MATCH-10 | 완료 화면은 `같은 카드 다시 하기`, `새 카드로 하기`, `4지선다로 가기`, `처음으로`를 제공한다. |

## 9. 접근성 및 반응형 요구사항

- 문서 언어는 `ko`로 선언하고 논리적인 제목 구조와 랜드마크를 사용한다.
- 세트 선택은 `fieldset`/`legend` 또는 동등한 그룹 이름을 제공한다.
- 카드와 보기는 네이티브 `button`을 사용하고 키보드 `Tab`, `Shift+Tab`, `Enter`, `Space`로 조작할 수 있다. 퀴즈 보기는 숫자 `1`~`4`도 지원한다.
- 짝맞추기 카드는 공개된 내용을 접근 가능한 이름에 포함한다.
- 정답 피드백과 완료 알림은 적절한 라이브 영역으로 전달하되, 반복적으로 화면 읽기를 방해하지 않는다.
- 포커스 표시를 제거하지 않으며 배경 대비가 분명한 3px 이상의 포커스 링을 제공한다.
- 본문과 조작 요소는 WCAG 2.2 AA 색 대비를 목표로 한다.
- 모바일의 모든 핵심 터치 대상은 최소 44×44px이다.
- 200% 확대와 320px 너비에서도 내용 손실이나 양방향 스크롤이 없어야 한다.
- `prefers-reduced-motion: reduce`에서는 흔들림과 불필요한 전환을 제거한다.
- 자세한 시각 규칙은 [DESIGN_SYSTEM.md](./DESIGN_SYSTEM.md)를 따른다.

## 10. 성공 기준

MVP는 외부 분석 도구를 넣지 않으므로 다음 기준을 자동 테스트와 수동 사용성 점검으로 판단한다.

### 정량 기준

- 대상 데이터 200자 및 등급별 `50/75/75` 수량 검증 통과
- 생성한 모든 4지선다 문제에서 보기 4개, 고유 표시값 4개, 정답 1개 보장
- 핵심 순수 로직 단위 테스트 통과율 100%
- 린트, 타입 검사, 테스트, 프로덕션 빌드 모두 성공
- 320px, 768px, 1280px 대표 너비에서 가로 넘침 0건
- 자동 접근성 검사에서 critical/serious 위반 0건을 목표로 함

### 사용성 기준

- 처음 보는 사용자가 설명 없이 세트 선택과 게임 시작을 완료한다.
- 키보드만으로 두 모드의 한 세트를 끝내고 결과 화면에 도달한다.
- 사용자가 현재 진행량과 정답 여부를 색상 없이도 말할 수 있다.
- 손상된 로컬 저장 값을 넣고 새로고침해도 흰 화면이 되지 않는다.

## 12. 제약과 위험

- 원문은 외부 위키 문서이므로 표 구조나 표기가 바뀔 수 있다. 수집 시점과 원문 표기를 보존하고 코드에는 검증된 정적 스냅샷을 포함한다.
- 복수 음훈은 보기를 모호하게 만들 수 있다. 원문 전체를 `sourceLabel`에 보존하되 대표 `eumhun`을 학습 표시값으로 사용하고, 문제 생성 시 충돌 항목을 제외한다.
- 로컬 기록은 브라우저·기기별이며 삭제될 수 있다. 이를 계정 기록처럼 표현하지 않는다.
- 정적 클라이언트 앱이므로 서버 장애는 없지만 JavaScript 비활성 환경에서는 게임을 제공할 수 없다. 기본 문서에는 앱 목적과 로딩 상태를 남긴다.
