# 한자랑 디자인 시스템

> 원본 기준: 저장소 루트의 `design.md`
> 상태: MVP 구현 기준
> 작성일: 2026-07-11

> 구현 일치 메모: 실제 CSS는 `--color-primary`, `--color-primary-bright`, `--color-primary-deep`, `--color-primary-soft`을 사용한다. 아래 문서의 이전 `--color-action*` 표기는 이 네 토큰에 각각 대응한다. 실제 페이지 컨테이너는 데스크톱 최대 1180px, 학습 화면은 최대 960px이다.

## 1. 전환 방향

원본의 흰색·옅은 회색·짙은 잉크색 표면, Electric Blue 중심 강조색, 8px 리듬, 부드러운 카드와 선명한 버튼의 대비, 절제된 그림자를 유지한다. HP 로고·가격·구매·상품 사진·할인·고객 후기처럼 판매 맥락에 속한 요소는 제거한다.

학습 앱의 시각 목표는 다음 세 문장으로 요약한다.

1. **차분하지만 심심하지 않다.** 넓은 여백과 높은 명암 대비 위에 파란 행동 요소를 제한적으로 사용한다.
2. **한자와 음훈이 가장 먼저 읽힌다.** 장식보다 글자 크기, 행간, 카드 상태를 우선한다.
3. **학생을 존중한다.** 유아용 캐릭터·과한 보상 효과·상업용 카탈로그 문법을 사용하지 않는다.

다크 모드는 MVP 범위가 아니다. 운영체제 설정과 무관하게 밝은 학습 표면을 제공한다.

## 2. 기본 원칙

- 한 화면의 강한 파란 요소는 원칙적으로 주요 행동 하나와 선택/진행 신호 하나 정도로 제한한다.
- 페이지 표면은 `canvas → subtle → ink`의 세 단계 안에서 구성한다.
- 학습 카드는 16px, 버튼·선택 컨트롤은 4px 모서리를 사용한다.
- 정답·오답은 색, 테두리, 기호, 문구를 항상 함께 사용한다.
- 그림자는 정보 그룹인 카드와 완료 패널에만 사용하고, 일반 섹션은 색상 대비와 선으로 구분한다.
- 사진과 삽화는 MVP에 넣지 않는다. 장식이 필요하면 CSS로 만든 짧은 각진 획 하나를 시작 영역에만 쓴다.
- 모든 핵심 조작은 44×44px 이상이며 키보드 포커스가 명확해야 한다.

## 3. 토큰 사용 방식

구현 토큰은 `app/globals.css`의 `:root`에 CSS 사용자 정의 속성으로 선언한다. 컴포넌트는 원시 hex·px 값을 반복하지 않고 아래 토큰을 참조한다. 의미가 같은 Tailwind 테마나 JavaScript 상수를 따로 만들어 두 개의 진실 원천을 만들지 않는다.

### 3.2 타이포그래피

#### 글꼴

```css
--font-ui: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI",
  "Noto Sans KR", "Apple SD Gothic Neo", "Malgun Gothic", sans-serif;
--font-hanja: "Noto Serif CJK KR", "Source Han Serif K", "Batang",
  "AppleMyungjo", serif;
```

- UI와 한국어 음훈은 `--font-ui`를 사용한다. 플랫폼 기본 글꼴을 먼저 사용해 외부 네트워크 요청과 글꼴 로딩 이동을 피한다.
- 문제 한자와 공개된 한자 카드에는 `--font-hanja`를 사용한다. 명조 계열은 큰 크기에서 획의 시작과 끝을 구별하기 쉽다.
- 두 스택 모두 한자 glyph가 없을 때 플랫폼 CJK 글꼴로 자연스럽게 fallback한다.
- Forma DJR Micro는 독점 서체이고 한국어·한자 범위를 보장하지 않으므로 사용하지 않는다.

#### 크기와 행간

| CSS 토큰 | 값 | 용도 |
|---|---|---|
| `--text-hanja` | `clamp(4.5rem, 14vw, 8rem)` / `1` | 퀴즈의 한자 한 글자 |
| `--text-display` | `clamp(2.25rem, 6vw, 3.5rem)` / `1.08` | 시작 화면 제목 |
| `--text-heading-lg` | `clamp(1.75rem, 4vw, 2.25rem)` / `1.2` | 화면 제목, 결과 점수 |
| `--text-heading-md` | `1.5rem` / `1.3` | 모드 카드·섹션 제목 |
| `--text-heading-sm` | `1.25rem` / `1.35` | 패널 제목, 공개 한자 카드 |
| `--text-body-lg` | `1.125rem` / `1.55` | 도입 설명, 주요 피드백 |
| `--text-body` | `1rem` / `1.5` | 본문, 버튼, 보기 |
| `--text-small` | `0.875rem` / `1.45` | 진행·메타 정보 |
| `--text-caption` | `0.75rem` / `1.4` | 출처와 제한적 보조 문구 |

CSS 구현에서는 크기와 행간 변수를 나누거나 같은 컴포넌트 선언에 함께 적용해도 된다. 본문은 400, 제목은 600~700, 버튼은 700을 사용한다. 원본의 영문 대문자·자간 규칙은 한국어 버튼 가독성에 맞지 않으므로 적용하지 않는다. 한국어 UI의 기본 자간은 `0`이다.

### 3.3 간격

8px를 기본 단위로 하고 4px 반 단계를 허용한다.

| 토큰 | 값 | 대표 용도 |
|---|---:|---|
| `--space-1` | 4px | 아이콘과 짧은 라벨 사이 |
| `--space-2` | 8px | 작은 내부 간격, 모바일 그리드 |
| `--space-3` | 12px | 버튼 세로 여백, 촘촘한 목록 |
| `--space-4` | 16px | 모바일 페이지 여백, 기본 gap |
| `--space-5` | 20px | 작은 패널 안쪽 |
| `--space-6` | 24px | 카드 안쪽, 데스크톱 gutter |
| `--space-8` | 32px | 큰 패널 안쪽, 섹션 내 묶음 |
| `--space-12` | 48px | 모바일 섹션 간격 |
| `--space-16` | 64px | 데스크톱 섹션 간격 |
| `--space-20` | 80px | 시작 화면의 큰 리듬에만 제한 사용 |

### 3.4 모서리, 선, 그림자

| 토큰 | 값 | 용도 |
|---|---:|---|
| `--radius-none` | 0 | 각진 획 장식 |
| `--radius-control` | 4px | 버튼, 보기, 등급 선택 |
| `--radius-small` | 8px | 진행 배지, 피드백 내부 요소 |
| `--radius-card` | 16px | 모드 카드, 게임 카드, 결과 패널 |
| `--radius-pill` | 9999px | 진행 막대와 작은 상태 점에만 사용 |
| `--border-hairline` | 1px | 기본 경계 |
| `--shadow-card` | `0 2px 8px rgba(26, 26, 26, 0.08)` | 모드·게임·결과 카드 |
| `--shadow-overlay` | `0 8px 24px rgba(26, 26, 26, 0.12)` | 실제 모달이 생길 때만 사용 |

버튼을 8px 이상 둥글게 만들지 않는다. 카드에 `--shadow-card`와 진한 테두리를 동시에 써서 과도하게 무겁게 만들지 않는다.

### 3.5 움직임

| 토큰 | 값 | 용도 |
|---|---:|---|
| `--duration-fast` | 120ms | pressed·포커스 보조 변화 |
| `--duration-normal` | 180ms | 카드 공개, 패널 등장 |
| `--ease-standard` | `cubic-bezier(0.2, 0, 0, 1)` | 기본 전환 |

- 불일치 카드는 약 700ms 동안 내용을 보여 주지만, 회전 자체는 180ms 이내다.
- 정답 축하를 위해 반복 튕김, 파티클, 자동 스크롤을 사용하지 않는다.
- `prefers-reduced-motion: reduce`에서는 모든 비필수 전환을 `0.01ms`로 줄이고 `transform` 기반 카드 회전을 제거한다.

## 4. 레이아웃

### 4.1 컨테이너

- 페이지 컨테이너: `width: min(calc(100% - 48px), 1180px)` (모바일은 좌우 16px)
- 데스크톱 좌우 여백: 최소 24px
- 게임 집중 영역: 최대 800px
- 퀴즈 문제·결과 영역: 최대 720px
- 주요 섹션 간격: 모바일 48px, 데스크톱 64px

원본의 1366px 카탈로그 폭은 많은 상품을 나열할 때 필요했지만, 학습 앱에서는 시선 이동이 커지므로 1180px로 제한한다. 게임 집중 영역은 960px로 더 좁게 제한한다.

### 4.2 화면별 구조

#### 시작 화면

1. 제품명과 한 문장 설명
2. 등급 선택 그룹과 실제 글자 수
3. 두 개의 모드 카드
4. 최근 학습 요약(기록이 있을 때)
5. 짙은 잉크색의 짧은 사용 안내·출처 밴드

모드 카드는 768px 이상에서 2열, 그 아래에서 1열이다. 첫 화면을 일반 대시보드처럼 사이드바와 작은 위젯으로 분할하지 않는다.

#### 짝맞추기

- 상단에 홈 이동, 현재 범위, 맞춘 쌍/전체, 시도 횟수를 둔다.
- 12장 그리드는 480px 미만 3열, 480px 이상 4열을 기본으로 한다.
- 그리드 최대 너비는 약 760px이며 중앙 정렬한다.
- 카드는 `aspect-ratio`를 사용하되 음훈이 두 줄까지 자연스럽게 보이도록 최소 높이를 보장한다.

#### 4지선다

- 진행 정보 → 큰 한자 문제 → 보기 → 피드백/다음 동작 순서다.
- 보기는 모바일 1열, 768px 이상 2×2 배치다.
- 응답 후 레이아웃이 크게 움직이지 않도록 피드백 공간을 문제 패널 안에 둔다.

#### 결과

- 점수와 정답률을 첫 번째 정보로 보여 준다.
- 오답 목록은 한자, 대표 음훈, 항목 구분선을 포함하는 단순 목록이다.
- 재시작 행동은 가장 관련 있는 하나만 파란 버튼으로, 나머지는 잉크 또는 outline 버튼으로 둔다.

## 5. 반응형 기준

| 구간 | 화면 변화 |
|---|---|
| `<480px` | 16px 페이지 여백, 모드 1열, 짝 카드 3열, 보기 1열, 장식 획 숨김 |
| `480~767px` | 짝 카드 4열 가능, 제목 크기 완만히 증가, 동작 버튼은 필요 시 전폭 |
| `768~1023px` | 모드 2열, 보기 2열, 패널 24~32px 안쪽 여백 |
| `>=1024px` | 1180px 컨테이너, 64px 섹션 리듬, 게임 영역 폭 제한 |
| `>=1280px` | 콘텐츠 폭은 더 늘리지 않고 주변 여백만 증가 |

- 긴 음훈은 잘라내지 않고 `overflow-wrap: anywhere` 또는 자연스러운 한국어 줄바꿈을 허용한다.
- 텍스트를 읽기 위해 가로 스크롤하거나 카드를 확대해야 하는 구조를 만들지 않는다.
- 고정 높이는 버튼·진행 막대 외에는 피하고 `min-height`를 사용한다.

## 6. 컴포넌트 명세

### 6.1 `app-header`

- 흰 배경, 하단 `--color-border` 1px, 높이 최소 64px
- 왼쪽 제품명 `한자랑`, 게임 중에는 오른쪽 또는 다음 줄에 현재 범위 표시
- 홈 이동은 실제 버튼/링크이며 44px hit area 보장
- 로그인, 검색, 장바구니, 햄버거 메뉴는 만들지 않는다.

### 6.2 `grade-selector`

- 보이는 제목 `어떤 한자를 연습할까요?`와 단일 선택 그룹
- 네 버튼은 기본 흰 배경/강한 경계, 선택 시 `--color-action-soft` 배경 + `--color-action` 2px 테두리 + `선택됨` 상태
- 각 라벨은 등급명과 글자 수를 함께 표시할 수 있다.
- pill 형태 대신 4px 모서리를 사용해 주요 컨트롤의 선명함을 유지한다.

### 6.3 `mode-card`

- 흰 패널, 16px 모서리, 24~32px padding, `--shadow-card`
- 모드 제목, 한 문장 설명, 학습 단위 안내, 시작 버튼 순서
- 카드 전체를 클릭 영역으로 만들지 않고 명시적인 시작 버튼을 제공한다.
- 한 카드의 주요 버튼만 파란색이 되도록 화면 전체 강조 밀도를 조절할 수 있다.

### 6.4 버튼

#### `button-primary`

- `--color-action` 배경, 흰 글자, 높이 최소 44px, 좌우 24px, 4px 모서리, 700 weight
- pressed: `--color-action-pressed`
- disabled: muted 표면 + secondary 텍스트 + `not-allowed`; opacity만 낮추지 않음

#### `button-secondary`

- 잉크 배경, 흰 글자, 동일한 크기 규칙
- 잉크 밴드 위에서는 흰 배경/잉크 글자의 역상 variant 사용 가능

#### `button-outline`

- 흰 배경, 잉크 또는 action 텍스트, 같은 색 1px 테두리
- 다시 하기·다른 모드·홈처럼 주요 행동 다음의 선택에 사용

모든 버튼은 `:focus-visible`에서 3px `--color-focus` 링과 2px offset을 갖는다. hover는 마우스 보조 상태일 뿐이며 hover 없이도 의미가 완전해야 한다.

### 6.5 `progress-header`

- 보이는 텍스트 `3 / 10문제`, `2 / 6쌍`, `시도 5번`을 제공한다.
- 진행 막대는 8px 높이, muted 트랙, action 채움, pill 모서리다.
- `<progress>` 또는 적절한 `role="progressbar"`와 현재/최대 값을 제공한다.
- 퍼센트만 표시해 전체 분량을 숨기지 않는다.

### 6.6 `match-card`

| 상태 | 시각·동작 |
|---|---|
| 가림 | 잉크 또는 옅은 회색 표면, `카드 열기` 힌트, 내용 미노출 |
| 공개 | 흰 표면, action 2px 테두리, 한자 또는 음훈 표시 |
| 확인 중 | 공개 상태 유지, 추가 입력 잠금 |
| 맞춤 | success-soft 배경, success 테두리, `✓ 맞춤` 문구 |
| 비활성 | 클릭 불가지만 내용과 맞춤 상태는 계속 보임 |

- 뒷면 버튼 이름은 `3번 카드, 열기`처럼 내용과 종류를 누설하지 않는다.
- 한자는 명조 계열 40~56px, 음훈은 산세리프 16~20px로 표시한다.
- 카드 종류를 뒷면의 색만으로 구분하지 않는다. 처음에는 두 종류 모두 같은 뒷면이다.

### 6.7 `quiz-prompt`와 `choice-button`

- 문제 한자는 `--text-hanja`, `--font-hanja`, `--color-text-strong`을 사용한다.
- 보기 버튼은 최소 높이 56px, 왼쪽 정렬, 16px 이상 글자, 4px 모서리다.
- 기본: 흰 배경 + strong border
- 선택 정답: success-soft + success 2px + `✓ 정답`
- 선택 오답: error-soft + error 2px + `! 선택한 답`
- 오답 응답 뒤 실제 정답 보기에도 success 테두리와 `정답` 문구를 표시한다.
- 답한 뒤 모든 보기를 비활성화하지만 opacity로 읽기 어렵게 만들지 않는다.

### 6.8 `feedback-banner`

- 정답은 success-soft, 오답은 error-soft, 복구 안내는 warning-soft 배경
- 상태 제목, 대표 음훈 설명, 필요한 다음 동작 순서
- `aria-live="polite"`를 사용하고 동일한 메시지를 불필요하게 반복하지 않는다.
- 색상과 이모지만 있는 피드백은 금지한다.

### 6.9 `completion-panel`

- 16px 모서리, 흰 패널, 32px padding, 제한적 `--shadow-card`
- 결과 제목, 핵심 수치, 복습 목록, 행동 그룹 순서
- 만점과 일반 완료 모두 지나친 축하 애니메이션 없이 명확하고 따뜻한 한국어를 사용한다.

### 6.10 `empty-error-panel`

- 빈 데이터: 무엇이 비었는지와 범위 재선택 동작
- 데이터 오류: `학습 자료를 불러오지 못했어요`와 홈/다시 시도 동작
- 저장 오류: 게임을 막지 않는 warning 스타일 안내
- 기술 스택, 예외 메시지, JSON 내용을 사용자에게 노출하지 않는다.

## 7. 상태와 상호작용 규칙

| 상태 | 필수 표현 |
|---|---|
| default | 명확한 라벨과 충분한 경계 |
| hover | 표면 또는 경계의 가벼운 변화; 의미를 새로 추가하지 않음 |
| pressed | action-pressed 또는 잉크 강도 변화 |
| focus-visible | 3px focus 링 + 2px offset |
| selected | 배경 + 2px 테두리 + 상태 속성/문구 |
| disabled | 읽을 수 있는 muted 스타일 + 실제 `disabled` |
| correct | success 색 + `✓` + `정답/맞춤` |
| incorrect | error 색 + `!` + `아쉬워요/선택한 답` |
| loading | 앱 이름과 짧은 준비 문구; 무한 장식 스켈레톤 금지 |
| complete | 결과 제목과 다음 행동; 자동으로 새 라운드 시작 금지 |

## 8. 접근성 규칙

- 일반 텍스트 4.5:1, 큰 텍스트 3:1 이상의 대비를 목표로 한다.
- `--color-action` 위에는 흰 텍스트만 사용하고 `--color-action-soft` 위에는 잉크/action 진한 텍스트를 쓴다.
- 포커스 outline을 `none`으로 제거하지 않는다.
- 카드와 보기의 DOM 순서는 시각 순서와 같아야 한다.
- 카드 회전의 앞·뒷면이 보조기술에 동시에 읽히지 않게 숨김 상태를 관리한다.
- 결과 화면 전환 시 결과 제목으로 포커스를 이동하되 사용자의 다음 행동을 자동 실행하지 않는다.
- 진행 상태에는 숫자 텍스트를 함께 제공한다.
- 모바일과 확대 환경에서 터치 대상 사이 간격을 최소 8px 확보한다.
- 고대비 모드에서도 테두리와 텍스트가 사라지지 않도록 시스템 색 강제 조정 여부를 확인한다.

## 9. 문구 원칙

- 짧고 직접적인 존댓말 또는 일관된 해요체를 사용한다.
- `오답입니다`보다 `아쉬워요. 정답은 하늘 천이에요.`처럼 다음 학습 정보를 준다.
- 버튼은 행동을 구체적으로 쓴다: `다음 문제`, `새 카드로 하기`, `처음으로`.
- `확인`, `진행`, `제출`처럼 맥락 없는 단어를 피한다.
- 유아적인 감탄사, 과도한 느낌표, 경쟁·벌점 표현을 피한다.
- 원문 표기는 출처/상세 정보에 보존하고 게임에서는 대표 음훈을 일관되게 표시한다.

## 11. 해야 할 것과 피할 것

### 해야 할 것

- 한 화면에서 가장 중요한 학습 행동을 하나 분명히 강조한다.
- 한자와 음훈을 충분한 크기와 여백으로 보여 준다.
- 카드 16px, 버튼 4px의 두 단계 모서리 체계를 유지한다.
- 흰색과 옅은 회색 섹션을 교대로 사용하고 짙은 밴드는 짧게 마무리에 쓴다.
- 구현에 새 시각 값이 필요하면 먼저 의미 토큰을 추가하고 문서와 CSS를 함께 갱신한다.
- 실제 긴 음훈과 320px 화면으로 레이아웃을 검증한다.

### 피할 것

- HP 로고, 셰브론 쌍, 상품 사진, 가격표, 할인 배지를 재사용하지 않는다.
- 파란색을 모든 카드 제목·아이콘·배경에 반복하지 않는다.
- 정답·오답을 초록/빨강만으로 표현하지 않는다.
- 카드마다 무거운 그림자, 광택, gradient, 3D 장식을 더하지 않는다.
- 버튼을 pill로 만들거나 한국어 라벨을 영문식 대문자·과한 자간으로 처리하지 않는다.
- 한 화면에 12장보다 많은 짝맞추기 카드를 배치하지 않는다.
- 사용자 입력 없이 자동으로 다음 문제나 새 라운드로 넘어가지 않는다.
